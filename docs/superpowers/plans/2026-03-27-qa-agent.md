# QA Agent 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 实现智能探索型 QA Agent，包含 4 个 skills 和测试框架

**Architecture:** 遵循 PM/Engineer Agent 的 skill 定义模式（Markdown + YAML frontmatter），每个 skill 独立可用

**Tech Stack:** Markdown, YAML, Playwright

---

## 文件结构

**创建文件**:
- `agents/qa/README.md` - Agent 入口和 skill 注册表
- `agents/qa/skills/exploratory-tester/skill.md` - 探索测试 skill
- `agents/qa/skills/spec-based-tester/skill.md` - 规范测试 skill
- `agents/qa/skills/bug-analyzer/skill.md` - Bug 分析 skill
- `agents/qa/skills/regression-suite/skill.md` - 回归测试 skill
- `agents/qa/test/exploratory-tester/evals/evals.json` - 探索测试用例
- `agents/qa/test/spec-based-tester/evals/evals.json` - 规范测试用例

**修改文件**:
- `skills-lock.json` - 注册 4 个新 skills
