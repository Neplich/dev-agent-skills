# Color Palettes

Use product-aware semantic palettes. Hex values are recommendations, not code tokens.

## Enterprise Analytics Dashboard

- Primary: `#1E40AF` - trust blue for main actions and active navigation.
- Secondary: `#334155` - neutral slate for secondary surfaces.
- Accent: `#D97706` - amber for highlights and attention.
- Background: `#F8FAFC` - low-noise workspace background.
- Surface: `#FFFFFF` - cards, tables, panels.
- Text primary: `#0F172A` - high contrast.
- Text secondary: `#475569` - supporting labels.
- Border: `#CBD5E1` - visible but quiet structure.
- Success: `#15803D`, Warning: `#B45309`, Error: `#B91C1C`, Info: `#2563EB`.

## Fintech

- Primary: `#0F172A`; Accent: `#0369A1`; Background: `#F8FAFC`.
- Use success/warning/error carefully and pair color with text or icons.

## Healthcare

- Primary: `#0F766E`; Accent: `#2563EB`; Background: `#F8FAFC`.
- Favor calm status colors and clear error recovery.

## E-commerce

- Primary: brand-specific; Accent: `#EA580C`; Success: `#15803D`.
- Use contrast for CTA hierarchy and purchase feedback.

## Consumer Social

- Primary: brand-specific vibrant color; Background: warm or neutral.
- Limit vivid colors to 3-4 roles so content remains readable.

## Color Rules

- Text contrast must meet WCAG AA: 4.5:1 for normal text, 3:1 for large text and UI boundaries.
- Status colors must not carry meaning alone; pair with labels or symbols in the design spec.
- Avoid default purple-blue AI gradients unless the brand explicitly owns that direction.
