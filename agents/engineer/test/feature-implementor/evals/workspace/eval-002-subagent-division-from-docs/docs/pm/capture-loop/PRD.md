# Capture Loop 队列重试 PRD

## 背景

Capture Loop 会接收客户端事件并写入本地处理队列。当前队列在处理器临时失败时只记录失败状态，缺少可控重试，导致短暂错误也需要人工干预。

## P0 用户故事

| ID | 用户故事 | 验收标准 |
| --- | --- | --- |
| US-001 | 作为维护者，我希望 Capture Loop 在处理器临时失败时自动重试，以减少人工恢复成本。 | 当处理器返回临时失败时，任务应进入 `retry_scheduled` 状态，并记录下一次重试时间。 |
| US-002 | 作为维护者，我希望重试次数有上限，以避免坏任务无限循环。 | 当任务达到最大重试次数后，应进入 `failed` 状态，并保留最后一次错误信息。 |
| US-003 | 作为维护者，我希望重试行为可被测试覆盖，以便后续修改不会破坏恢复路径。 | 应新增或更新队列服务测试，覆盖首次失败、达到上限和成功恢复场景。 |

## 非目标

- 不引入新的持久化数据库。
- 不增加后台调度器实现。
- 不改变 Capture Loop 的外部 API。
