# 角色授权模型

## 当前状态

系统提供 `guest`、`user` 和 `admin` 三种角色。服务端必须以可信身份记录确定当前角色，客户端输入不得提升角色。

## 已确认权限边界

| 资源或操作 | guest | user | admin |
| --- | --- | --- | --- |
| 查看公开页面 | 允许 | 允许 | 允许 |
| 查看本人资料 | 禁止 | 允许 | 允许 |
| 查看管理端审计日志 | 禁止 | 禁止 | 允许 |
| 管理用户角色 | 禁止 | 禁止 | 允许 |

所有 `/admin/*` 请求都必须在服务端完成身份认证和 `admin` 角色校验。审查范围仅包含角色识别、管理端资源授权和关键授权路径，不改变现有角色定义。

## 验收标准

- 非 `admin` 请求无法读取管理端审计日志或管理用户角色。
- 修改请求头、查询参数或请求体中的角色值不能提升权限。
- 安全审查须给出可定位的代码证据、影响、严重度和修复及回归验证建议。
