# 会话管理

## 当前状态

登录成功后，系统使用服务端会话保护用户资料和账户设置。匿名用户不能访问这些资源。

## 已确认会话边界

- 会话标识必须不可预测，且只通过安全 Cookie 传递。
- 登录成功时必须创建新会话，不能复用登录前会话标识。
- 会话必须在 30 分钟无活动后过期。
- 用户退出后，服务端会话必须立即失效，旧标识不得继续访问已认证资源。

审查范围包含登录、会话解析、过期和退出路径，不包含 MFA 或密码策略调整。

## 验收标准

- 攻击者不能枚举或预测其他用户的会话标识。
- 过期会话和已退出会话均被受保护端点拒绝。
- 安全审查须给出可定位的代码证据、影响、严重度和修复及回归验证建议。
