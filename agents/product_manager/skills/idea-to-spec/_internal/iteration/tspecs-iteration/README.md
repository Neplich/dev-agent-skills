# TEST_SPEC Iteration (tspecs-iteration)

## Overview

Update an existing test specification while preserving traceability, coverage
history, and versioning.

## When to use

- `tspecs-validator` found issues
- PRD / TRD / API changes require test updates
- QA review found missing coverage or outdated cases

## Quick start

```text
Update this test spec to match the new auth flow and fill the validator gaps.
```

## Related skills

- `tspecs-gen`
- `tspecs-validator`
- `trace-check`
